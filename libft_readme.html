01 — description

What is this?

The very first project at 42. Before you use a function, you write it.

libft is a hand-rolled implementation of the most common functions from libc — string manipulation, memory operations, character checks — plus a set of utility functions not in the standard library at all.

The result is a .a static library you link into every project that follows. It grows with you: add functions as you need them. By the time you finish 42, libft is yours.

The rule that makes it interesting: no #include <string.h>, no shortcuts, no calling the real functions. Every byte, every loop, written by hand.

◈

Part 1 — libc

Re-implementations of standard C functions. Same signature, same behavior.

◈

Part 2 — extra utils

Functions not in libc but useful everywhere: ft_split, ft_itoa, ft_substr…

◈

Bonus — linked lists

Full singly-linked list API: new, add, delete, iterate, map, clear.

◈

Static library

Compiled into libft.a. Link with -L. -lft in every future project.

03 — architecture

How it works

One Makefile, one archive. Every future project just links it in.

The build pipeline

build.diagram

  ft_memset.c  ──► cc -Wall -Wextra -Werror -c ──► ft_memset.o
  ft_split.c   ──► cc -Wall -Wextra -Werror -c ──► ft_split.o

       ⋮                                                    ⋮
       │                                                    │
       └──────────────────────────────────────────────────┘
                            │
                            ▼

                            │
                            ▼
                       libft.a   ← link this into any future project

Using libft in another project

linking.diagram
  # your future project's Makefile

  LIBFT_PATH  = ../libft
  LIBS        = -L$(LIBFT_PATH) -lft

  $(NAME): $(OBJS)
      make -C $(LIBFT_PATH)
      cc $(OBJS) $(LIBS) -o $(NAME)

  # in any .c file
  #include "libft.h"

  ft_putstr_fd("hello\n", 1);   // works
  ft_split("a:b:c", ':');       // works
  ft_itoa(42);                  // works

The t_list struct (bonus)

t_list.h
  typedef struct s_list
  {
      void          *content;   // any data — you decide the type
      struct s_list *next;     // pointer to the next node, NULL if last
  }   t_list;

  // example: building a list
  t_list *head = ft_lstnew("first");
  ft_lstadd_back(&head, ft_lstnew("second"));
  ft_lstadd_back(&head, ft_lstnew("third"));
  // head → "first" → "second" → "third" → NULL
  ft_lstclear(&head, free);   // clean up
04 — instructions

Getting started

Clone, make, link. Nothing more.

Build

$ git clone https://github.com/mostafaloka/libft.git
$ cd libft
$ make # compiles libft.a (Part 1 + Part 2)
$ make bonus # also compiles bonus linked-list functions

Makefile targets

$ make # build libft.a
$ make bonus # build libft.a including bonus functions
$ make clean # remove .o files
$ make fclean # remove .o files + libft.a
$ make re # fclean then make

Quick test with a main

// test.c
#include "libft.h"
#include <stdio.h>
 
int main(void)
{
    printf("%zu\n", ft_strlen("hello")); // 5
    printf("%s\n", ft_itoa(-2147483648)); // -2147483648
    char **arr = ft_split("a:b:c", ':');
    printf("%s %s %s\n", arr[0], arr[1], arr[2]); // a b c
    return (0);
}
 
$ cc test.c -L. -lft -o test && ./test
